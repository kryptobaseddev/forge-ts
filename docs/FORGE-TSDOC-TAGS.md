# forge-ts TSDoc Tag Reference

> Canonical reference for every TSDoc tag recognized, enforced, or generated by forge-ts.
> This document is the single source of truth for the forge-ts tag system.

## Tag Status Legend

- **ACTIVE** — Implemented: parsed by walker, used by enforcer/generator
- **PLANNED** — Designed but not yet implemented in code

---

## Standard TSDoc Tags

Tags defined by the [@microsoft/tsdoc](https://tsdoc.org) specification. forge-ts recognizes all 29 standard tags via `@microsoft/tsdoc-config` integration.

### API Contract Tags

| Tag | Kind | Layer | Status | Enforced By | Purpose |
|-----|------|-------|--------|-------------|---------|
| `@param` | block | API | ACTIVE | E002 | Document function/method parameters |
| `@returns` | block | API | ACTIVE | E003 | Document return value |
| `@throws` | block | API | ACTIVE | W002, guide-discovery | Document thrown exceptions |
| `@typeParam` | block | API | ACTIVE | E015 | Document generic type parameters |
| `@defaultValue` | block | API | ACTIVE | E014 | Document default value for optional properties |

**Examples:**

```typescript
/**
 * Fetch a user by their unique identifier.
 *
 * @param id - The user's UUID
 * @returns The resolved user object
 * @throws {NotFoundError} When no user matches the given ID
 * @typeParam T - The user model type to return
 * @example
 * ```typescript
 * const user = await fetchUser<AdminUser>("abc-123");
 * ```
 */
export async function fetchUser<T extends User>(id: string): Promise<T> { ... }
```

```typescript
/**
 * Maximum retry count before failing.
 * @defaultValue 3
 */
retries?: number;
```

## forge-ts Custom Tags

Tags defined by the `@forge-ts/tsdoc-config` preset. These extend the TSDoc standard with forge-ts-specific semantics.

### API Generation Tags

| Tag | Kind | Layer | Status | Used By | Purpose |
|-----|------|-------|--------|---------|---------|
| `@route` | block | API | ACTIVE | OpenAPI gen | HTTP method and path for REST endpoint documentation |
| `@response` | block | API | PLANNED | OpenAPI gen | HTTP response type and status code |
| `@query` | block | API | PLANNED | OpenAPI gen | Query parameter documentation for REST endpoints |
| `@header` | block | API | PLANNED | OpenAPI gen | HTTP header parameter documentation |
| `@body` | block | API | PLANNED | OpenAPI gen | Request body schema documentation |

**Examples:**

```typescript
/**
 * Retrieve all users with optional filtering.
 *
 * @route GET /api/users
 * @query limit - Maximum number of results (default: 50)
 * @query offset - Pagination offset (default: 0)
 * @header Authorization - Bearer token
 * @response 200 - Array of User objects
 * @response 401 - Unauthorized
 * @response 500 - Internal server error
 *
 * @param req - Express request object
 * @returns Paginated user list
 */
export async function getUsers(req: Request): Promise<UserListResponse> { ... }
```

```typescript
/**
 * Create a new project resource.
 *
 * @route POST /api/projects
 * @body CreateProjectInput - The project creation payload
 * @response 201 - Created project
 * @response 400 - Validation error
 */
export async function createProject(req: Request): Promise<Project> { ... }
```

## Enforcement Rule Coverage by Tag

Every tag in the forge-ts system has a clear enforcement story — either an active rule validates it, a planned rule will validate it, or it's used for generation/filtering (no enforcement needed).

### Tags Enforced by Rules

| Tag | Rule | Default Severity | Enforcement |
|-----|------|-----------------|-------------|
| Summary (implicit) | E001 | error | Every exported symbol must have a summary |
| `@param` | E002 | error | Every function parameter must be documented |
| `@returns` | E003 | error | Non-void functions must document return value |
| `@example` | E004 | error | Exported functions must have executable examples |
| `@packageDocumentation` | E005 | error | Package entry points must have package overview |
| Class members | E006 | error | Public class members must be documented |
| Interface members | E007 | error | Interface properties must be documented |
| `{@link}` | E008 | error | Link targets must exist in the symbol graph |
| `@remarks` | E013 | error | Public functions/classes must have detailed remarks |
| `@defaultValue` | E014 | warn | Optional properties should document defaults |
| `@typeParam` | E015 | error | Generic type parameters must be documented |
| `@public`/`@beta`/`@internal` | E016 | error | Every export must declare its release stage |
| `@deprecated` | W003 | warn | Deprecation must include explanation |
| `@deprecated` (cross-pkg) | W004 | warn | Importing deprecated symbols from other packages |
| `@see` | W005 | warn | Symbols with {@link} should also have @see |
| TSDoc syntax | W006 | warn | All TSDoc must parse without syntax errors |
| FORGE:AUTO references | W007 | warn | Guide references must point to existing symbols |
| Guide coverage | W008 | warn | Public symbols should appear in at least one guide |
| `{@inheritDoc}` | W009 | warn | PLANNED — Source must exist and have content |
| `@breaking` + `@migration` | W010 | warn | PLANNED — Breaking changes need migration path |
| `@since` | W011 | warn | PLANNED — New exports should have version annotation |

### Tags Enforced by Config Guards

| Tag/Config | Rule | Enforcement |
|------------|------|-------------|
| tsconfig.json flags | E009 | Required strict-mode flags must be true |
| forge-ts rule severities | E010 | Rules cannot be weakened below locked level |
| biome.json rules | E011 | Biome rules cannot be weakened below locked level |
| package.json engines | E012 | Node.js version cannot be downgraded below minimum |
| `@internal` re-export | E017 | PLANNED — Internal symbols must not leak through public barrels |
| `@response` on routes | E018 | PLANNED — Route handlers should document response types |

## tsdoc.json Preset

The `@forge-ts/tsdoc-config` package ships this preset. All projects using forge-ts inherit it via:

```json
{
  "$schema": "https://developer.microsoft.com/json-schemas/tsdoc/v0/tsdoc.schema.json",
  "extends": ["@forge-ts/tsdoc-config/tsdoc.json"]
}
```

## Interoperability

The forge-ts tsdoc.json preset is consumed by:

| Tool | Integration | What It Gets |
|------|-------------|-------------|
| **eslint-plugin-tsdoc** | Automatic via `tsdoc.json` | Recognizes all custom tags — no `tsdoc-undefined-tag` errors |
| **TypeDoc** | `"extends": ["@forge-ts/tsdoc-config/tsdoc.json"]` | Consistent tag recognition in generated docs |
| **API Extractor** | `"extends": ["@forge-ts/tsdoc-config/tsdoc.json"]` | Consistent tag trimming behavior |
| **forge-ts check** | `TSDocConfigFile.loadForFolder()` | W006 surfaces syntax errors using this config |
| **VS Code** | Via TypeScript IntelliSense | Hover docs respect tag structure |
