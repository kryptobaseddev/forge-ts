## AuditEvent

A single audit event recorded in the forge-ts audit trail.

| Property    | Type                      | Required | Description                                                  |
| ----------- | ------------------------- | -------- | ------------------------------------------------------------ |
| `timestamp` | `string`                  | Yes      | ISO 8601 timestamp of when the event occurred.               |
| `event`     | `AuditEventType`          | Yes      | Discriminated event type.                                    |
| `user`      | `string`                  | Yes      | OS username of the actor (falls back to "unknown").          |
| `reason`    | `string \| undefined`     | No       | Mandatory for lock/unlock/bypass events; optional otherwise. |
| `details`   | `Record<string, unknown>` | Yes      | Event-specific payload.                                      |

## BypassConfig

Configuration for the bypass budget system.

| Property        | Type     | Required | Description                                                          |
| --------------- | -------- | -------- | -------------------------------------------------------------------- |
| `dailyBudget`   | `number` | Yes      | Maximum number of bypasses allowed per calendar day. Default: 3      |
| `durationHours` | `number` | Yes      | Duration in hours before a bypass automatically expires. Default: 24 |

| Property    | Type     | Required | Description                                                                |
| ----------- | -------- | -------- | -------------------------------------------------------------------------- |

| `createdAt` | `string` | Yes      | ISO 8601 timestamp when the bypass was created.                            |

## RuleSeverity

Severity level for an individual enforcement rule. - `"error"` — violation fails the build. - `"warn"`  — violation is reported but does not fail the build. - `"off"`   — rule is disabled entirely.

RuleSeverity
