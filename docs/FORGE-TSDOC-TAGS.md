## Standard TSDoc Tags

Tags defined by the [@microsoft/tsdoc](https://tsdoc.org) specification. forge-ts recognizes all 24 standard tags via `@microsoft/tsdoc-config` integration.

### API Contract Tags

| Tag | Kind | Layer | Status | Enforced By | Purpose |

| `@throws` | block | API | ACTIVE | W002, guide-discovery | Document thrown exceptions |
| `@typeParam` | block | API | ACTIVE | E015 | Document generic type parameters |
| `@defaultValue` | block | API | ACTIVE | E014 | Document default value for optional properties |

**Examples:**

```typescript

 * ```typescript
 * const user = await fetchUser<AdminUser>("abc-123");

### Enforcement Control Tags

| Tag | Kind | Layer | Status | Used By | Purpose |
|-----|------|-------|--------|---------|---------|
| `@forgeIgnore` | modifier | Dev | ACTIVE | enforcer | Skip all enforcement rules on this symbol |

The `@forgeIgnore` tag is a modifier tag that suppresses all enforcer diagnostics for the annotated symbol. Use it sparingly for symbols that intentionally break documentation conventions (e.g., auto-generated code, re-exports with passthrough semantics).

**Examples:**

```typescript
/**
 * Re-exported for backwards compatibility. See the upstream package.
 * @forgeIgnore
 * @public
 */
export { legacyHelper } from "./compat.js";
```

```typescript
/**
 * Auto-generated type -- documentation lives in the schema file.
 * @forgeIgnore
 * @internal
 */
export type GeneratedPayload = z.infer<typeof PayloadSchema>;
```

For file-level ignores, see the `enforce.ignoreFile` configuration below.

## Enforcement Rules (33 total)

forge-ts ships 33 enforcement rules: 20 error-level (E001-E020) and 13 warning-level (W001-W013).

### TSDoc Documentation Rules

These rules validate TSDoc presence and correctness on exported symbols.

| Code | Key | Default | Condition |
|------|-----|---------|-----------|
| E001 | `require-summary` | error | Exported symbol is missing a TSDoc summary comment |
| E002 | `require-param` | error | Function/method parameter lacks a `@param` tag |
| E003 | `require-returns` | error | Non-void function/method lacks a `@returns` tag |
| E004 | `require-example` | error | Exported function/method is missing an `@example` block |
| E005 | `require-package-doc` | error | Package entry point (index.ts) is missing `@packageDocumentation` |
| E006 | `require-class-member-doc` | error | Public/protected class member is missing a TSDoc comment |
| E007 | `require-interface-member-doc` | error | Interface/type alias property is missing a TSDoc comment |
| E008 | -- | error | Dead `{@link}` reference -- target symbol does not exist in the project |
| E013 | `require-remarks` | error | Exported function/class is missing a `@remarks` block |
| E014 | `require-default-value` | warn | Optional property of interface/type is missing `@defaultValue` |
| E015 | `require-type-param` | error | Generic symbol is missing `@typeParam` for a type parameter |
| E016 | `require-release-tag` | error | Exported symbol is missing a release tag (`@public`, `@beta`, or `@internal`) |
| E017 | `require-internal-boundary` | error | `@internal` symbol re-exported through public barrel (index.ts) |
| E018 | `require-route-response` | warn | `@route`-tagged function missing `@response` tag |
| E019 | `require-no-ts-ignore` | error | Non-test file contains `@ts-ignore` or `@ts-expect-error` directive |
| E020 | `require-no-any-in-api` | error | Exported symbol has `any` in its public API signature |
| W001 | -- | warn | TSDoc comment contains parse errors (documented; subsumed by W006) |
| W002 | -- | warn | Function body throws but has no `@throws` tag (documented; reserved) |
| W003 | -- | warn | `@deprecated` tag is present without explanation text |
| W004 | -- | warn | Importing a deprecated symbol from another package |
| W005 | `require-see` | warn | Symbol references other symbols via `{@link}` but has no `@see` tags |
| W006 | `require-tsdoc-syntax` | warn | TSDoc parser-level syntax error (invalid tag, malformed block, etc.) |
| W007 | `require-fresh-guides` | warn | Guide FORGE:AUTO section references a symbol that no longer exists |
| W008 | `require-guide-coverage` | warn | Exported public symbol is not mentioned in any guide page |
| W009 | `require-inheritdoc-source` | warn | `{@inheritDoc}` references a symbol that does not exist |
| W010 | `require-migration-path` | warn | `@breaking` tag present without `@migration` path |
| W011 | `require-since` | warn | New public export missing `@since` version tag |
| W012 | `require-fresh-link-text` | warn | `{@link}` display text appears stale relative to target summary |
| W013 | `require-fresh-examples` | warn | `@example` block may be stale -- function call arg count mismatches signature |

**Notes on E008, W001-W004:**
E008, W001, W002, W003, and W004 are emitted by the enforcer but are not part of the `EnforceRules` config interface. They cannot be individually set to `"off"` via `enforce.rules`. They always emit (W-codes as warnings, E-codes as errors) unless suppressed by `@forgeIgnore` or `enforce.ignoreFile`. When `enforce.strict` is `true`, all W-codes are promoted to errors.

### Config Guard Rules

These rules protect project configuration from regression. They are not per-symbol rules and cannot be suppressed with `@forgeIgnore`.

| Code | Key | Default | Condition |
|------|-----|---------|-----------|
| E009 | -- | error | tsconfig.json required strict-mode flag is missing or disabled |
| E010 | -- | error | Config drift: a rule severity is weaker than the locked value in `.forge-lock.json` |
| E011 | -- | error | Biome config weakening: a biome.json rule severity dropped below locked level |
| E012 | -- | error | package.json `engines.node` version below minimum or required field missing |

Config guard rules can be bypassed via the bypass budget system (`forge-ts bypass E009`).

## Enforcement Suppression

### Per-Symbol: `@forgeIgnore` Tag

Add the `@forgeIgnore` modifier tag to any symbol's TSDoc to skip all enforcement rules on that symbol.

```typescript
/**
 * @forgeIgnore
 * @public
 */
export type SkippedType = unknown;
```

### Per-File: `enforce.ignoreFile`

Set `enforce.ignoreFile` in your config to point to a file listing symbol names to skip (one per line). Lines starting with `#` are comments.

```typescript
// forge-ts.config.ts
export default defineConfig({
  enforce: {
    ignoreFile: ".forge-ignore",
  },
});
```

```text
# .forge-ignore
# Generated types -- docs live in the schema
GeneratedPayload
GeneratedResponse
# Knip-detected dead exports
legacyAdapter
```

This integrates with [Knip](https://knip.dev) for dead-export detection:

```bash
npx knip --reporter json | jq -r '.exports[].name' > .forge-ignore
```

### Per-Rule: `enforce.rules`

Individual rules (those with a key in the table above) can be set to `"error"`, `"warn"`, or `"off"`:

```typescript
// forge-ts.config.ts
export default defineConfig({
  enforce: {
    rules: {
      "require-example": "warn",      // Downgrade E004 to warning
      "require-remarks": "off",       // Disable E013 entirely
      "require-fresh-guides": "off",  // Disable W007
    },
  },
});
```

### Strict Mode

When `enforce.strict` is `true`, all warnings are promoted to errors:

```typescript
export default defineConfig({
  enforce: {
    strict: true,  // W003, W005, W006, etc. all become errors
  },
});
```

## Per-Group Enforcement (tsdoc.enforce)

The `tsdoc.enforce` config section controls enforcement severity by TSDoc standardization group. This allows bulk severity control over categories of tags without configuring each rule individually.

> **Note:** Per-group enforcement is actively being implemented. The config interface is defined in `ForgeConfig.tsdoc.enforce` and will be wired into the enforcer pipeline. The individual `enforce.rules` overrides always take precedence over group-level settings.

| Group | Tags Covered | Default |
|-------|-------------|---------|
| `core` | `@param`, `@returns`, `@remarks`, `@deprecated`, `@packageDocumentation`, `@privateRemarks`, `{@link}`, `{@label}` | `"error"` |
| `extended` | `@example`, `@throws`, `@see`, `@decorator`, `@defaultValue`, `@typeParam`, `@eventProperty`, `@readonly`, `@override`, `@sealed`, `@virtual`, `{@inheritDoc}` | `"warn"` |
| `discretionary` | `@public`, `@beta`, `@internal`, `@alpha` | `"off"` |

```typescript
// forge-ts.config.ts
export default defineConfig({
  tsdoc: {
    enforce: {
      core: "error",
      extended: "warn",
      discretionary: "off",
    },
  },
});
```

## TSDoc Config Ownership

forge-ts **owns** `tsdoc.json`. The config flow:

```
forge-ts.config.ts (tsdoc section)
        |
   tsdoc.json (written by forge-ts init / writeConfig)
        |
   consumed by: eslint-plugin-tsdoc, TypeDoc, API Extractor
```

The walker loads TSDoc config via `TSDocConfigFile.loadForFolder()` from `@microsoft/tsdoc-config`. Configs are cached per directory. If no `tsdoc.json` is found, falls back to bare `new TSDocConfiguration()`.

## Interoperability

The forge-ts tsdoc.json preset is consumed by:

| Tool | Integration | What It Gets |
|------|-------------|-------------|
| **eslint-plugin-tsdoc** | Automatic via `tsdoc.json` | Recognizes all 15 custom tags -- no `tsdoc-undefined-tag` errors |
| **TypeDoc** | `"extends": ["@forge-ts/tsdoc-config/tsdoc.json"]` | Consistent tag recognition in generated docs |
| **API Extractor** | `"extends": ["@forge-ts/tsdoc-config/tsdoc.json"]` | Consistent tag trimming behavior |
| **forge-ts check** | `TSDocConfigFile.loadForFolder()` | W006 surfaces syntax errors using this config |
| **VS Code** | Via TypeScript IntelliSense | Hover docs respect tag structure |
| **Knip** | Via `enforce.ignoreFile` | Dead exports excluded from enforcement |
